# DocMind 기능 테스트 가이드

> 브라우저 직접 테스트용. Phase 2~8(코어)까지 구현된 기능을 사용자 관점에서 기능별로 점검한다.
> 진행 상태/설계는 [IMPLEMENTATION_PLAN.md](IMPLEMENTATION_PLAN.md), 함정은 [../CLAUDE.md](../CLAUDE.md) §8 참조.

## 0. 사전 준비 (필수)

**터미널 2개를 띄운다:**

```bash
# 터미널 1
INNGEST_DEV=1 pnpm dev
# 터미널 2 (1번이 :3000 뜬 뒤)
pnpm inngest          # 'apps synced' 뜨면 연결 완료
```

**확인 체크리스트:**

- `.env.local`에 `ANTHROPIC_API_KEY`, `VOYAGE_API_KEY`, `SUPABASE_*`, Google Auth 키 설정
- ⚠️ **Voyage 결제수단 등록 필수** — 무료 티어는 3 RPM이라 인터뷰·생성 중 즉시 429로 막힌다 (CLAUDE.md §8 함정)
- DB 시드: `pnpm db:seed` (이미 했으면 생략 — "Penta Security" 워크스페이스)
- 브라우저: http://localhost:3000 → `@pentasecurity.com` 구글 계정으로 로그인
- (선택) Slack/Email 실제 발송 테스트 시 `SLACK_BOT_TOKEN`+`SLACK_DEFAULT_CHANNEL_ID`, `RESEND_API_KEY` 설정

## 1. 인증 (로그인)

- **테스트**: 로그아웃 상태로 `/agent` 등 직접 접근 → `/login`으로 리다이렉트되는지
- 구글 로그인 → 홈으로 진입
- **기대**: `pentasecurity.com` 도메인만 허용 (다른 도메인 계정은 거부)
- 우상단에 내 이메일 + 로그아웃 버튼 표시

## 2. 홈 (`/`)

- **기대**: 6종 문서 카드(영업 제안서/기획안/사업 계획서/기술 문서/회의 자료/마케팅 자료)
- 카드 클릭 → 인터뷰 화면으로 진입(문서 생성 + 인터뷰 세션 시작)

## 3. 지식 베이스 (`/kb`) — _Demo A의 사전 단계_

- **URL 등록**: URL 탭 → `https://pentasecurity.com/products/wapples` 입력 → "URL 등록"
  - **기대**: 카드가 `crawling` 칩으로 추가 → 자동 새로고침되며 잠시 후 `ready`로 전환, 제목·요약·태그 채워짐
  - 백그라운드에서 크롤→청크→임베딩이 도는 중 (터미널 2 inngest 로그에 함수 실행 보임)
- **파일 업로드**: 파일 탭 → PDF/PPTX/DOCX/XLSX 드래그 → 동일하게 `ready`까지
- **카드 클릭** → 우측 Sheet: 원본 텍스트 미리보기, 학습 청크 수, 마지막 감지 시각, **"지금 재학습"** 버튼
- **흔한 함정**: `error` 칩이면 크롤 실패(접근 불가 URL/지원 안 되는 형식) — 터미널 로그 확인

## 4. 5질문 인터뷰 + 생성 (`/chat/[id]`) — _Demo A 핵심_

- 홈에서 "영업 제안서" 카드로 시작
- **5문답 진행** (데모 권장값): 독자=**임원**, CTA=**계약**, 반론=**가격**, 소스=**WAPPLES**, 분량=**10**
- **기대 화면 요소**:
  - 상단 **진행도 트랙**(5단계)이 답할 때마다 채워짐
  - 각 질문에 **Quick Reply 버튼** 2~4개 (클릭 또는 직접 입력)
  - "소스 자동 감지" **InsightBox** — KB에서 매칭된 청크 요약 미리보기
- 마지막 답변 후 자동으로 생성 → `/deck/[versionId]`로 이동
- **기대 성능**: 생성 ~25초 내 (LLM 다중 호출)

## 5. 미리보기 & 다운로드 (`/deck/[versionId]`)

- **기대**: `<Deck>` 슬라이드 미리보기(표지~CTA), Penta 브랜드 토큰 적용
- **".pptx 다운로드"** 클릭 → 실제 PowerPoint 파일 다운로드 → 열어서 미리보기와 일치하는지
- **흔한 함정**: 한글 폰트 깨짐(시청 PC에 폰트 없을 때) — 데모 PC는 맑은 고딕 기본

## 6. 문서함 (`/docs`)

- **목록**: 생성한 문서가 카드 그리드로. 상단 **유형/상태 필터 칩** 클릭 → 필터링되는지
- 카드 클릭 → **상세(`/docs/[id]`)**:
  - 좌측 **버전 타임라인**(v1, v2… 상태 칩/변경노트/생성일 + 미리보기·.pptx·"최신과 비교" 링크)
  - 우측 **버전 비교 diff** (기본 최신↔직전, `+추가`/`−삭제` 색상)
  - **참고**: 새로 만든 문서는 v1뿐이라 "비교할 이전 버전이 없습니다"가 정상. diff를 보려면 같은 문서를 재생성해 v2를 만들거나, 아래 7번 에이전트 루프로 v2(드래프트)를 생성

## 7. 에이전트 자율 루프 + 실시간 피드 (`/agent`) — _Demo B 핵심_

화면 구성: 좌측 에이전트 목록 / 중앙 탭(활동 피드·루프 구조·생성된 문서) / 우측 통계 4종 + 승인 큐.

⚠️ **중요**: "지금 감지" 버튼은 등록된 URL 소스를 재크롤해 **내용이 5% 이상 바뀐 경우에만** 루프를 시작한다. KB가 그대로면 아무 일도 안 일어나는 게 정상이다. 그래서 변경을 **강제**해야 한다.

**7-A. 실시간 5단계 피드 관찰 (권장)**

1. 브라우저에서 `/agent` 열고 **"활동 피드" 탭**을 띄워둠 (새로고침 X)
2. **터미널 3**에서:
   ```bash
   pnpm verify:agent
   ```
3. **기대**: 실행 직후 활동 피드에 `감지 → 인식 → 판단 → 행동 → 학습`이 **~1초 간격으로 새로고침 없이** 한 줄씩 추가됨 (이게 SSE 실시간 push). "루프 구조" 탭에선 현재 단계 원이 **펄스**. 우측 통계(오늘 자동 실행/갱신 문서)도 갱신
4. 단, 이 스크립트는 끝에서 **자동 승인**까지 해버리므로 승인 카드는 잠깐 떴다 사라진다

**7-B. 승인 클릭을 직접 체험하려면**

- `verify:agent`는 자동 승인하므로 수동 클릭 테스트엔 부적합하다. "발행 승인" 버튼을 직접 누르려면 **자동 승인 없이 변경만 트리거하는 헬퍼**가 필요하다 → `scripts/trigger-change.ts`(해시 강제 + detect 이벤트만, 승인 X)를 추가하면:
  1. `pnpm trigger-change` 실행
  2. `/agent` 우측 **승인 큐에 카드 등장** (대기 중)
  3. **"발행 승인"** 클릭 → 토스트 + 피드에 `approval.decided` + "생성된 문서" 탭에 발행됨 칩

## 8. 알림 (Slack / Email) — _7-B 승인 시점에 발송_

- Slack/Resend 토큰이 설정돼 있으면, **"발행 승인" 클릭 시** `#docmind-demo`에 Block Kit 메시지 + 워크스페이스 멤버에게 메일 발송
- 메시지의 **"발행 승인 보기" 버튼** 클릭 → `/agent?approval=<id>`로 진입 → 해당 카드 **하이라이트**
- 토큰 미설정 시: 토스트는 "발행 승인됨"만(발송 skipped), 동작엔 문제 없음
- 발송 실패 시: 경고 토스트 + 활동 피드에 **빨간 칩**(`notification.failed`)

## 9. 스케줄 / Mode C (`/schedules`)

- **등록**: cron(예 `* * * * *` = 매분) + 문서 템플릿(유형/제목/독자/CTA/반론/소스/분량) 입력 → "스케줄 등록"
- **기대**: 1분 안에 새 문서가 자동 생성되어 `/docs`에 나타남 (터미널 2 inngest 로그에 `agent.generate.scheduled` 실행)
- ⚠️ **매분 생성은 LLM 비용·문서 누적** → 확인 후 목록에서 **"비활성화" 또는 "삭제"** 할 것

## 10. 설정 (`/settings`)

- **에이전트 정책**: "자동 실행" 토글 / "발행 정책" manual·auto / "Slack 알림 채널" 입력 → **저장**
- **기대**: 저장 후 새로고침해도 값 유지. (auto로 바꾸면 이후 7번 루프에서 승인 없이 자동 발행되는지 확인 가능 — 테스트 후 manual로 되돌리기 권장)
- 이메일 수신자(멤버 목록), 브랜드 템플릿(읽기), Slack/Email **연결 상태 배지** 표시

## 11. 디자인 (Phase 8 진행 중)

- **폰트**: 전 화면이 **Pretendard**로 렌더되는지 (이전 Geist 대비 한글 자연스러움)
- **팔레트**: 버튼이 **보라색 primary**, 배경 따뜻한 off-white, 테두리 hairline 톤
- **다크 모드**: ⚠️ **토글 UI는 아직 미연결**(Phase 8 단계4). 미리 보려면 브라우저 devtools에서 `<html>`에 `class="dark"` 추가 → 배경/텍스트가 다크로 전환되는지

## 빠른 점검 순서 (10분 데모 루트)

1. 로그인 → 2. `/kb`에서 WAPPLES URL 등록(`ready` 대기) → 3. 홈 "영업 제안서" → 5문답 → 미리보기 → .pptx → 4. `/docs` 확인 → 5. `/agent` 열고 `pnpm verify:agent`로 실시간 5단계 → 6. (헬퍼 추가 시) 수동 승인 → 7. `/schedules` 매분 등록 후 `/docs`에 자동 생성 확인 → 비활성화
